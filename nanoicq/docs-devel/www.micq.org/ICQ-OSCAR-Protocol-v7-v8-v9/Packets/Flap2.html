SNAC

Channel: 2

The data sent with a channel 2 FLAP.

Parameters

Data type Content Name Description
WORD.B xx xx FAMILY The family this command belongs to.
WORD.B xx xx COMMAND The command to perform.
WORD.B xx xx FLAGS Flags accompanying this command. Usually zero.
In ICQv8 bit 15 of the flags means that extra data comes before the body of the SNAC. Not fully understood, it takes the format WORD [length] DATA, where DATA is [length] bytes long.
DWORD.B xx xx xx xx REFERENCE This field is used to associate data from different packets. If the client starts two seperate searches, this field is used to differentiate the search results returned from the server. For non-reply packets coming from the server, this field is usually a random number, if more than one non-reply packet is sent by the server in succession, the number is merely incremented. Non-reply packets from the client will always use the command value in this field. Packets from the server or client that are replies to other packets will have the field from that packet copied here.

The highest bit in this DWORD has special significance (probably because they use a signed int on the server-side), take care not to overflow into the highest bit if you are simply incrementing your request ID for every packet. You should mask this bit out and cycle within the 31-bit range of numbers.
DATA ... DATA The data this SNAC contains.

Examples

00 01 00 03 00 00 A8 6B 3E B2    Snac: Type x01/x03, ID xA86B3EB2, Flags 0
