#define CAPABILITIES

Here are the capabilities that I encountered.

CAP_AIM_VOICE {09461341-4C7F-11D1-8222-444553540000}
AIM specific capability.
CAP_AIM_SENDFILE {09461343-4C7F-11D1-8222-444553540000}
AIM specific capability.
CAP_AIM_ISICQ {09461344-4C7F-11D1-8222-444553540000}
The client is, in fact, an ICQ client, and not an AIM client. This capability may occur as a type-2 message's capability: then the content will contain the sender's dc info.
CAP_AIM_IMIMAGE {09461345-4C7F-11D1-8222-444553540000}
AIM specific capability.
CAP_AIM_BUDDYICON {09461346-4C7F-11D1-8222-444553540000}
AIM specific capability.
CAP_AIM_STOCKS {09461347-4C7F-11D1-8222-444553540000}
AIM specific capability.
CAP_AIM_GETFILE {09461348-4C7F-11D1-8222-444553540000}
AIM specific capability.
CAP_AIM_SERVERRELAY {09461349-4C7F-11D1-8222-444553540000}
The client supports messages in the style of direct messages (aka type-2 messages). If this capability is not set, the server will refuse type-2 messages to this contact with this capability as the messages' capability.
CAP_AIM_GAMES {0946134A-4C7F-11D1-8222-444553540000}
AIM specific capability
CAP_AIM_SBUD {0946134B-4C7F-11D1-8222-444553540000}
AIM specific capability.
CAP_AIM_INTEROPERATE {0946134D-4C7F-11D1-8222-444553540000}
Setting this lets AIM users receive messages from ICQ users, and ICQ users receive messages from AIM users. Windows AIM 5.2.3139 is the only thing I know that sets this. Well, and Gaim 0.60cvs.
Also, though you advertise the capability, it does not get sent to other users with your other capabilities.
CAP_UTF8 {0946134E-4C7F-11D1-8222-444553540000}
The ICQ client supports UTF-8 messages. Each outgoing message that uses the UTF-8 character set has CAP_UTF8 appended as a GUID. Otherwise, the character set used is totally undefined. Note that CAP_UTF8 also signals UCS-2BE encoded & tagged type-1 messages are understood.
CAP_RTFMSGS {97B12751-243C-4334-AD22-D6ABF73F1492}
The ICQ client supports RTF type-2 messages. Each outgoing message that is in RTF format has CAP_RTFMSGS appended as a GUI.
CAP_STR_2001 {A0E93F37-4C7F-11D1-8222-444553540000}
ICQ 2001 sends this to all contacts upon (visible) login inside an otherwise empty CAP_SERVERRELAY type-2 message.
CAP_STR_2002 {10CF40D1-4C7F-11D1-8222-444553540000}
ICQ 2002 sends this the same way as CAP_STR_2001 inside an otherwise empty CAP_SERVERRELAY type-2 message.
CAP_IS_2001 {2E7A6475-FADF-4DC8-886F-EA3595FDB6DF}
Capability sent by ICQ 2001.
CAP_IS_WEB {563FC809-0B6f-41BD-9F79-422609DFA2F3}
Sent by ICQ Lite and ICQ2go.
CAP_LICQ {09491349-4C7F-11D1-8222-444553540000}
Note the 0x49 instead of 0x46 to CAP_AIM_SERVERRELAY. This is sent by licq; probably a typo.
CAP_AIM_CHAT {748F2420-6287-11D1-8222-444553540000}
AIM specific capability.
CAP_TRILL_CRYPT {F2E7C7F4-FEAD-4DFB-B235-36798BDF0000}
Trillian crypted message.
CAP_TRILL_UNK {97B12751-243C-4334-AD22-D6ABF73F1409}
Another unknown capability invented by Trillian.
CAP_MICQ {6d494351-20a9-2052-2e4b-2e20wwxxyyzz}
Sent by mICQ. The last four bytes determine the client version.
CAP_SIM {97B12751-243C-4334-AD22-D6ABF73F14xx}
Sent by SIM and Kopete. The last byte is the version number; if it is less than or 1<<6, then it's Kopete, otherwise it's SIM, with the byte being (major + 1) << 6 + minor. Kopete uses 1 for 0.6 up to 0.6.1b and 64 = 1<<6 for 0.6.2. Why can't those guys just use their own capability?
CAP_MACICQ {DD16F202-84E6-11D4-90DB-00104B9B4B7D}
Capability supposedly sent by Mac ICQ.
CAP_KXICQ {09491344-4c7f-11d1-8222-444553540000}
Sent by kxICQ2.
CAP_SIMNEW {53494d20-636c-6965-6e74-2020wwxxyyzz}
Sent by SIM >= 0.8.3. The last four bytes determine the client version. Interpreted in US-ASCII, the capability reads "SIM client ####".
